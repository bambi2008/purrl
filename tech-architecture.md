# Purrl — 技术架构框架（B）

> 2026-06-28 | 原则：全外部AI API，零自建GPU，独立开发者可hold

---

## 整体架构

```
┌─────────────────────────────────────────┐
│           前端（Web独立站 / PWA）          │
│   落地页 · 上传 · 我的猫 · 各支柱 · 支付     │
└────────────────┬────────────────────────┘
                 │ API
┌────────────────▼────────────────────────┐
│              后端 API 层                   │
│   用户/CatProfile/订单/订阅管理            │
└────┬───────────┬──────────┬──────────────┘
     │           │          │
┌────▼────┐ ┌───▼────┐ ┌───▼─────────┐
│ AI编排层 │ │ 数据库  │ │ 支付/POD     │
│ 队列+回调│ │+对象存储│ │ Stripe+履约  │
└────┬────┘ └────────┘ └─────────────┘
     │
┌────▼──────────────────────────────────┐
│  AI能力(全外部API,不自建GPU)            │
│  FAL(图) 可灵(视频) TripoSR(3D)        │
│  vision(色板) DeepSeek(孪生对话)       │
└───────────────────────────────────────┘
```

---

## 选型

| 层 | 选型 | 理由 |
|----|------|------|
| 前端 | Next.js (React) | SSR利于SEO(独立站靠搜索获客)，Vercel部署全球CDN |
| 后端 | Next.js API Routes / 独立Node | MVP前后端一体省事 |
| 数据库 | Supabase (Postgres) | 自带认证+存储+实时，独立开发者友好 |
| 对象存储 | Supabase Storage / Cloudflare R2 | R2无出口流量费 |
| AI编排 | 队列(异步)+Webhook回调 | LoRA/3D要几分钟必须异步 |
| 支付 | Stripe(海外)+微信/支付宝(国内) | Stripe已配OAuth |
| POD履约 | Printful(海外)/1688(国内) | 配饰/3D雕塑代发 |
| 部署 | Vercel(前端)+Supabase(后端) | 零运维按量付费 |

---

## 为什么全用外部AI API

```
不自建GPU：
  ① 验证期用量小，自建月烧几千起，浪费
  ② FAL/可灵模型比自建好
  ③ 按调用付费=成本随收入走，不亏

何时自建：日活过万+AI成本>自建时(远不到)
```

---

## MVP最小技术栈

```
Next.js + Supabase + FAL + 可灵 + Stripe + Printful
   前后端    数据认证   图   视频   收款    履约
        Vercel部署
```

一个独立开发者能hold住，无专门运维。

## 关键技术风险

| 风险 | 应对 |
|------|------|
| LoRA训练慢(几分钟) | 异步队列+基础模型占位 |
| 3D重建质量不稳 | Phase2前100张猫测失败率 |
| 国内访问Vercel慢 | 国内版用腾讯云/阿里云+CDN |
| AI API限流/涨价 | 多供应商备份(FAL+Replicate) |
