# Purrl — 数据流框架（C）

> 2026-06-28 | 技术地基：一张照片，一次提取，六支柱复用

---

## 核心原则

**一只猫一份档案（CatProfile），跨六支柱复用。** 不是六个独立工具，是一个猫档案驱动的六种呈现。护城河在数据层——猫档案越完整，用户迁移成本越高。

---

## 识美数据模型 CatProfile

```
CatProfile {
  // ① 身份层
  cat_id
  原始照片(原图 + 去背图)

  // ② 外观层(视觉提取，上传即做，秒级近免费)
  品种推测       暹罗/英短/布偶/田园...
  毛色色板       主色+辅色+花纹(Pantone级 3-5色值)
  毛长           长毛/短毛/无毛
  眼色           影响配饰配色
  花纹类型       纯色/虎斑/三花/奶牛...

  // ③ 体态层(姿态估计)
  体型           紧凑/修长/圆润
  姿势           坐/卧/站

  // ④ 气质层(AI判读)
  气质标签       优雅/呆萌/霸气/仙气

  // ⑤ 衍生层(按需预计算)
  3D网格         一次重建，供3D雕塑+旋转展示
  LoRA模型       一次训练，供写真集+短视频保脸
}
```

---

## 数据→六支柱映射

```
                  CatProfile (一次提取)
                        │
   ┌────────┬──────────┼──────────┬─────────┬────────┐
   ▼        ▼          ▼          ▼         ▼        ▼
 写真集   AI短视频   真猫配饰   3D雕塑   数字孪生  出行美学
```

| 支柱 | 用到的字段 |
|------|-----------|
| 写真集 | LoRA模型 + 气质标签 + 品种(选文化风格) |
| AI短视频 | LoRA模型 + 姿势 |
| 真猫配饰 | 毛色色板 + 眼色 → AI配色算法 |
| 3D雕塑 | 3D网格 + 姿势 |
| 数字孪生 | 气质标签 + 品种 → 生成猫性格 |
| 出行美学 | 毛色色板 + 体型 → 选包配色和尺寸 |

---

## 计算时机决策

| 数据 | 时机 | 理由 |
|------|------|------|
| 色板/气质/品种 | 上传即提取(秒级) | vision便宜，配饰推荐马上要用 |
| LoRA模型 | 首次上传后异步训练 | 训练几分钟，先用基础模型出图，好了再升级 |
| 3D网格 | 按需(点3D雕塑才建) | 重建贵(1-3元)，非每个用户都买雕塑 |

---

## 解决什么

```
❌ 错误：每个支柱各自处理照片 = 重复计算 + 数据不一致
✅ Purrl：上传一次 → 一份CatProfile → 全局共享 = 一次计算处处复用
```
